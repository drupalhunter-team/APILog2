Виды данных.

Линейные:

* Структура (одна запись).
* Массив структур (таблица).
* Связный список.

Связи между таблицами:

* "Foreign Key" Указатель на базовую запись.
* Список - указатель на первую базовую запись и счётчик
* Связный список
* Связь по значениям
* Связь по порядку
* Комбинации перечисленного

Визуализация данных

* Показ записи
* Показ линейной таблицы.
* Показ иерархии в избранном столбце.
	Указываем таблицы и список столбцов, способ связи между таблицами
	1. Список столбцов с указанием
		Имени столбца
		Список таблиц с указанием используемых в них столбцов
	2. Указать список столбцов, по которым будут связаны таблицы

Дерево 

Дизассемблер.

Цели:
	1. Показ дизассемблированного кода на экране.
	2. Ассемблер.
	3. Возможность работы трассировщика/эмулятора.

Внутренний формат дизассемблера:
	1. Желательно сходство с C++.
	2. Измельчение команд (Расчёт EA и прочее).
	
	Описание команды: 
		Команда.
		Операнды[].
		Вид - самостоятельная команда, часть EA, др.

	Описание операнда: 
		Вид - целый(i), знаковый(s), беззнаковый(u), с плавающей запятой(f).
		Размер.
		Расположение - константа, область регистров, область временных переменных
Примеры:

	ADD EAX, [EBX + 3]
	EAX = EAX + *(EBX + 3)
	t1[i32] = EBX[i32] + 3[i32]
	t2[i32] = *t1[i32]
	EAX[i32] = EAX[i32] + t2[i32]

	XCHG EDX, [EAX]
	t1 = *EAX[i32]; *EAX[i32] = EDX[i32]; EDX[i32] = t1

	XCHG EDX, [2 * EAX]
	t1[u32] = 2[u] * EAX[u32]; t2[i32] = *t1[u32]; *t1[u32] = EDX[i32]; EDX[i32] = t2

Перспективный внутренний формат:
	2. Контроль времени существования переменных.
